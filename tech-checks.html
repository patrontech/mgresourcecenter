Tech ChecksLast updated: September 15th, 2020

Tech Checks are the process in which an Event Tech determines if an attendee has their microphone, camera, and network connection configured properly prior to them joining their session with the talent.

Note

While there is a set of criteria that needs to be met prior to an approval, ultimately the question that needs to be answered as a 'yes' in order to pass is "Can this attendee have a one-to-two minute uninterrupted conversation with the talent without dealing with consistent or frequent audio/video complications?"

Prerequisites

  • Completed Getting Started

  • Shadowed an Event Operations Manager

  • Watched all instructional videos

  • Completed first proficiency test

Overview

Shown below is an illustration that walks you through the decision making process on how you can determine if an attendee passes or fails our Tech Check. Aside from this illustration that highlights some of the issues we run into, the rest of this walkthrough will demonstrate what the workflow is for an ideal Tech Check with an attendee.

GuideLast updated: September 15th, 2020

Initial Setup

Before beginning any Tech Checks be sure to follow the following setup steps:

  1. Make sure you are not in the room. When loading the page for a Meet and Greet you can determine if you are in the room or not by seeing if there is a button in the middle of the page that says "Click Here to Join Room".
  2. If you are in the room in your settings, change Autojoin so that it reads as Autojoin OFF. Once it's set to Autojoin OFF refresh the page and look for the "Click Here to Join Room" button to confirm that you've successfully exited the room.
  3. Turn off your microphone and camera. *These do not need to be enabled for you to perform a tech check.
  4. Once the Event Staff is in the room let them know, in the host chat, that you're there and are currently working through the customer queue to get everyone approved.

Starting a Tech Check

Every tech check begins by initiating a private session with the attendees that are waiting in the Customer Queue.

To initiate a private session begin by right clicking (do not left click, that will prompt you to let them into the room) on the attendee's name that you would like to check. You'll then click Ok to the prompt asking you if you'd like to begin the private session.

Once the private session modal launches, the customer will have be notified and given a 3 second warning that a tech check is about to begin.


Left shows the Event Tech's perspective and right shows the customer's.

Vetting an Attendee

Start off the conversation by using the following script:


“Hello [attendee name] I am [event tech name] with tech support. I'm just doing a quick check on your audio and video before you begin your one-on-one session. Can you hear me and see me okay?"


Once you've confirmed that you're able to hear and see the attendee and they can hear and see you..


"Okay great! You're all set, don't change anything and your session will be starting shortly."

You can now click Approve and then proceed to exit the Tech Check by clicking Close Session.

Determining a Pass or Fail

Like mentioned on the last page, there is a specific set of criteria that the customer has to meet in order to be considered as approved to have their session with the talent.

While there are requirements that cannot be looked past, there is a bit of discretion required from the Event Tech in order to make the final determination wether or not that attendee passes or fails a Tech Check.

First lets cover the specific criteria that I'm referencing:

  • You can clearly see the attendee
  • The attendee can clearly see you
  • You can clearly hear the attendee
  • The attendee can clearly hear you

Warning

No approval can occur until all four of the above questions can be answered with a confident YES.

In addition to that, there are circumstances where the answer to those four questions may be yes but the last determining factor of vetting an attendee is assessing the strength of network connection they have throughout your time in the Tech Check with them. Ultimately the question you should be asking yourself to make this call is:



"Can this attendee have a one-to-two minute uninterrupted conversation with the talent without dealing with consistent or frequent audio/video complications?"


These connectivity issues are shown in a variety of ways. A customers video could be completely frozen but you can hear them or the video consistently pauses for a few seconds, resumes, and then continues. There is no specific cadence or combination in which these poor connection side effects occur so we cannot anticipate every possibility but we do need to try and identify them when possible and ideally help the attendee fix those when we can.

The best way to think about this decision is to put yourself in the customer's shoes. Would you want to pay $X amount of money to speak with someone you're a fan of only to battle with attempting to understand or hear each other? Probably not.

And then for the talent, do you think it would be enjoyable for someone interacting with a fan to have someone's video be continually freezing or lagging behind the audio? I would assume not.

This isn't always the easiest determination to make at first but you will very quickly pick up on the signs that point to denying someone based on the above factors.

TroubleshootingLast updated: September 15th, 2020

Basic

While most attendees have no issue connecting to a reliable wireless network or getting their audio and video setup correctly, we do run into some people who are a bit less savvy and need some hand holding. Below you'll find some common things we've run into while trying to perform these tech checks.

Note

It's important to note that while we will try our best to support an attendee who is having a tough time getting setup but there are only so many things we can attempt before letting the event staff know that alternative arrangements will need to be made for that customer.

Private Session Never Begins

Every tech check begins by initiating a private session with the attendees that are waiting in the Customer Queue.

If you've began a private session but the attendee never joins it could mean a couple things:

  • The attendee has closed this page but they've not yet been removed from the customer queue.
  • The attendee is unable to obtain a strong enough connection to begin the private session.

Regardless of the reason of why the session won't load the following steps should be followed:

  1. Move on and continue checking any remaining guests in the queue.
  2. Once you've felt enough time has passed (~5 minutes) attempt to begin another private session with the attendee experiencing issues.
  3. Repeat this process 1 more time if this attempt is unsuccessful.
  4. If after the third attempt a connection can still not be made then please let the Event Staff know that the customer is unable to connect to complete a tech check and an alternative arrangement will need to be made for them.

Note

Because the attendee cannot connect we have no way to communicate with them so there aren't a lot of options for troubleshooting this issue.

The attendee cannot see or hear you

If the attendee cannot see you, hear you or any combination of the two there's not any specific settings or configuration troubleshooting that we can advise.

As usual, it's best to confirm that the attendee is using the browser based on our recommendation.

  • macOS (Apple): Google Chrome
  • Windows: Google Chrome
  • iOS: Safari
  • Android: Google Chrome

If the issue persists after the attendee is confirmed to be using an appropriate browser or changing to the correct browser doesn't fix the issue we should suggest for them to attempt this on a different device.

A computer is always the best option to suggest but we shouldn't deter the attendee from attempting any other device, having them switch from their phone to a tablet or tablet to a computer is usually the easiest route to a resolution and will often time fix the problem.

Network connectivity issues

When an attendee is experiencing a poor network connection we can only advise and hope they can resolve the issue on their end.

If an attendee is connected to WiFi:

Suggest that they connect directly to their router with an ethernet cable. If that's not an option have them try and move closer to where their router is located.

If an attendee is using a mobile device:

Ask them if they are connected to WiFi and if they're not if they're able to connect to a wireless network rather than using their cellular data.

When an attendee makes a change to their network connection you'll need to close the session, give them time to make that change, and then re-attempt the tech check after some time has passed.

Nothing is working, what do I do?

If after multiple attempts and different routes of troubleshooting have been attempted and we can still not resolve an issue with an attendee we need to make sure that the event staff is aware.

Post a brief message in the host chat letting the event staff know of the situation, something like this is a good example:


Hey, [event staff name] - [attendee first/last name] is having issues connecting due to the strength of their network. I've attempted to troubleshoot this a number of times now and it doesn't seem to be helping or getting better. At this point it would be best to offer an alternative solution for them. Let me know if you have any questions

There are simply some attendees we are not going to be able to accommodate and have to make that determination after we've given a reasonable amount of effort towards trying to help them.
